YAML configuration files define [Model, Resources, and Agent servers](/latest/about/concepts/core-components). Each server gets its own configuration block, giving modular control over the entire training environment.

## How Servers Connect

A training environment typically runs all three server types together. The agent server (the agent harness) config references model and resources servers by server ID. Those references wire the environment together — the agent knows which model to call and which resources to use.

## Config File Locations

Each server type has a dedicated directory with its implementations and their configs:

```text
# Model Server Config
responses_api_models/
  └── openai_model/
      └── configs/openai_model.yaml

# Resources Server Config
resources_servers/
  └── example_single_tool_call/
      └── configs/example_single_tool_call.yaml

# Agent Server Config
responses_api_agents/
  └── simple_agent/
      └── configs/simple_agent.yaml
```

## Server Block Structure

Each config file defines a server with this structure:
```yaml
server_id:                    # Your unique name for this server
  server_type:                # responses_api_models | resources_servers | responses_api_agents
    implementation:           # Directory name inside the server type directory
      entrypoint: app.py      # Python file to run
      # ... additional fields vary by server type
```

Server types have additional required fields (for example, `domain` for resources servers; `resources_server` and `model_server` for agents). See [Configuration](/latest/reference/configuration) for complete field specifications.

Config files often use the same name for both server ID and implementation:

```yaml
example_single_tool_call:        # ← Server ID
  resources_servers:
    example_single_tool_call:    # ← Implementation
```

These serve different purposes:

- **Server ID** (`example_single_tool_call` on line 1): Your identifier for this server instance. Used in API requests and when other servers reference it. Rename it to `my_weather` or `weather_prod` if you want.

- **Implementation** (`example_single_tool_call` on line 3): Must match the folder `resources_servers/example_single_tool_call/`. This selects which code to run.

Examples use matching names for simplicity, but the two values are independent choices.
