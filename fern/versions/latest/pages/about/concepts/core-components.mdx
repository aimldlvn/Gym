## What is an Environment?

An environment defines the task the agent accomplishes, the actions the agent takes, and the state of the world the agent observes and acts upon. The environment also determines how the agent's performance is evaluated — what counts as success and how reward is assigned.

These concepts map to three server components:

- **[Agent](/latest/reference/agent-server)** servers (the agent harness) determine whether a rollout is single-step or multi-step, single-turn or multi-turn, and orchestrate the full rollout lifecycle: calling the model, routing tool calls to resources, and collecting the final reward. The agent server does not run an LLM itself — it delegates all text generation to the model server.
- **[Model](/latest/reference/model-server)** servers (LLM inference endpoints) are stateless. They receive a conversation and return the model's next output (text, tool calls, or code) with no memory or orchestration logic.
- **[Resources](/latest/reference/resources-server)** servers (the environment backend) provide the tasks agents solve, the tools and external state they interact with, and the verification logic that scores performance and returns reward signals for training. Each resources server manages isolated per-rollout state via session IDs.

```
┌──────────────────────────────────────────┐
│              Agent Server                │
│                                          │
│  run():                                  │
│    1. resources.seed_session()  ─────────────►  Resources Server
│    2. multi-step/multi-turn loop:        │
│         model.responses()       ─────────────►  Model Server
│         resources.my_tool()     ─────────────►  Resources Server
│    3. resources.verify()        ─────────────►  Resources Server
│         → reward                         │
└──────────────────────────────────────────┘

┌───────────────────────────┐   ┌───────────────────────────────────┐
│       Model Server        │   │        Resources Server           │
│                           │   │                                   │
│  responses():             │   │  seed_session(): init env state   │
│    conversation           │   │  my_tool():      execute action   │
│    → text, tool calls,    │   │  verify():       evaluate → reward│
│      or code              │   │                                   │
└───────────────────────────┘   └───────────────────────────────────┘
```
