## Motivation

Building and scaling RL training environments for LLMs runs into four recurring problems:

- **Environments coupled to training**: Most RL workflows bind environment logic to the training pipeline, which blocks complex agent loops, slows iteration on environment design, and prevents controlled ablations.
- **No consistent trajectory schema**: Chat Completions dominates today, but it was designed for stateless, single-turn interactions. Agentic rollouts interleave reasoning, tool calls, and text across multiple turns. Without a native schema, every environment needs custom parsing and serialization.
- **Unmanaged external resources**: Environments depend on sandboxed execution, databases, and APIs. Each rollout needs isolated instances that initialize and clean up reliably.
- **Rollout collection that does not scale**: Training requires thousands of parallel rollouts. Environment instances need distribution, load balancing, and fault tolerance.

## NeMo Gym

[NeMo Gym](https://github.com/NVIDIA-NeMo/Gym) is an open-source library that builds RL environments and scales rollout collection, and integrates with any training framework.

NeMo Gym addresses the challenges above:

- **Decoupled architecture**: Environment development is fully separated from training, so teams build, test, and iterate on environments independently of the RL training loop. Interoperates with existing environments, systems, and RL training frameworks.
- **Environment scaffolding**: Patterns and infrastructure that accelerate environment development for multi-step, multi-turn, and user modeling scenarios.
- **Standardized trajectories**: The OpenAI Responses API is the native format, with a schema that represents multi-turn, tool-calling agentic rollouts without custom serialization.
- **Managed resource lifecycles**: Resources servers handle initialization, isolation, and cleanup of external dependencies (sandboxes, APIs, databases) per rollout.
- **Scalable rollout collection**: Distributes thousands of parallel rollouts with load balancing and fault tolerance.
- **Growing environment hub**: NVIDIA and community-contributed environments and datasets for training and evaluation.

<Tip>
The name "NeMo Gym" comes from historical reinforcement learning literature, where "Gym" refers to a collection of RL training environments.

</Tip>
