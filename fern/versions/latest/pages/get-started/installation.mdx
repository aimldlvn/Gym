Three install paths: **Source** (recommended), **PyPI** (library-only), **Container** (RL training with NeMo RL).

## Prerequisites

Before you start, verify the following:

1. Operating system matches a [verified configuration](/latest/get-started/prerequisites#verified-configurations).
2. Python 3.12+ is installed and on PATH.
3. You have at least 2 GB of free disk space for dependencies.
4. Optional: GPU drivers installed for local inference or RL training.

<Note title="See Also">
For comprehensive system requirements, refer to [Prerequisites](/latest/get-started/prerequisites).
</Note>

## Installation Methods

Pick the path that matches what you want to do.

| If you want to... | Use this path |
| --- | --- |
| Run benchmarks, evaluate models, follow the [Quickstart](/latest/get-started/quickstart) | **Source** â€” only path that ships the bundled benchmarks and `ng_list_benchmarks` |
| Use NeMo Gym as a library inside your own project | **PyPI** (note the limitation in the tab) |
| Train with NeMo RL on a GPU cluster | **Container** â€” the NGC image bundles NeMo RL |

<Tabs>

<Tab title="Source (Recommended)">

The Source path gives you the full set of bundled environments, benchmarks, and example data. The [Quickstart](/latest/get-started/quickstart) and [Create Your First Environment](/latest/get-started/create-first-environment) both assume this layout.

1. **Clone the repository**:

   ```bash
   git clone git@github.com:NVIDIA-NeMo/Gym.git
   cd Gym
   ```

2. **Create the virtual environment and install**:

   ```bash
   uv venv --python 3.12 && source .venv/bin/activate
   uv sync
   ```

3. **Install `uv` if you do not have it**:

   ```bash
   curl -LsSf https://astral.sh/uv/install.sh | sh
   source $HOME/.local/bin/env
   ```

</Tab>

<Tab title="PyPI">

The simplest path, but it does not yet ship the bundled benchmarks or `ng_list_benchmarks`.

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install nemo-gym
```

<Warning>
The current PyPI release lacks the bundled benchmarks and `ng_list_benchmarks`. To run the [Quickstart](/latest/get-started/quickstart) end-to-end, use the **Source** path until the next release.
</Warning>

</Tab>

<Tab title="Container">

For RL training with NeMo RL, pull the latest NGC container:

```bash
docker pull nvcr.io/nvidia/nemo-rl:v0.5.0.nemotron_3_super
```

<Note title="See Also">
For full container setup including GPU passthrough and volume mounts, refer to the [NeMo RL training tutorial](/latest/train-models/nemo-rl-grpo/setup).
</Note>

</Tab>

</Tabs>

## Installation Verification

Run these commands to confirm the install succeeded:

| Command | Success Check | If it fails |
| --- | --- | --- |
| `ng_help` | A table of `ng_*` commands prints | Activate the venv: `source .venv/bin/activate` |
| `ng_list_benchmarks` (Source only) | A table of about 45 benchmarks (`gpqa`, `aime25`, `ifbench`, and others) | You are on the PyPI path â€” switch to Source |
| `python -c "import nemo_gym; print(nemo_gym.__version__)"` | A version number prints | Reinstall: `uv sync` (Source) or `pip install nemo-gym` (PyPI) |

## Troubleshoot Installation

<Accordion title="Command not found: ng_run / ng_help">
Activate the virtual environment in the terminal you are using:

```bash
source .venv/bin/activate
```

Verify the install:

```bash
pip show nemo-gym
```
</Accordion>

<Accordion title="ng_list_benchmarks shows no benchmarks">
You are on the PyPI install path. The bundled benchmarks under `benchmarks/` only ship with the Source install. Switch to the Source path or wait for the next PyPI release.
</Accordion>

<Accordion title="uv sync fails on dependency resolution">
Use a fresh venv:

```bash
rm -rf .venv
uv venv --python 3.12 && source .venv/bin/activate
uv sync
```
</Accordion>

<Accordion title="ImportError: No module named 'nemo_gym'">
The package is not installed in the active environment. Verify and reinstall:

```bash
pip list | grep nemo-gym
uv sync   # Source path
# or
pip install nemo-gym   # PyPI path
```
</Accordion>

<Accordion title="Ray socket path error on cluster filesystem">
On systems with long working directory paths (for example, Lustre), Ray's AF_UNIX socket can exceed the 107-byte Linux limit. Set:

```bash
export RAY_TMPDIR=/tmp
```
</Accordion>
