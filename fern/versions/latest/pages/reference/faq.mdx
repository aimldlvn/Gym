## Concepts

<Accordion title="What's the difference between SFT and RL?">

SFT (supervised fine-tuning) trains on input + gold-response pairs. RL (reinforcement learning) trains on input + a scorer that rates the model's own responses, encouraging higher-scored ones. NeMo Gym is the source of training data and scorers for RL, and can also generate synthetic SFT data by running strong teacher models against the same environments.

For the long form, see [Training approaches](/latest/about/concepts/training-approaches).

</Accordion>

<Accordion title="What's a monotonic trajectory and why does NeMo RL require it?">

A trajectory is monotonic when the token sequence only grows across turns — earlier tokens are never edited or dropped. NeMo RL enforces this in the vLLM worker (replacing re-tokenized prefixes with the original token IDs) and in NeMo Gym postprocessing (asserting strict-increasing token IDs across turns).

Common ways to break monotonicity: chat templates that strip prior reasoning, agent harnesses that summarize history, sliding-window context management, environment-state pruning. Workarounds: keep all thinking in the chat template, use the non-thinking variant of the model, or audit your agent for context surgery.

</Accordion>

## CI and docs

<Accordion title="The build-docs CI check failed. What do I do?">

Reproduce locally with `fern docs dev` from the repo root. Common causes: broken Markdown links, invalid MDX syntax, missing required frontmatter on new pages.

</Accordion>
