Train NVIDIA [Nemotron Nano 9B v2](https://huggingface.co/nvidia/NVIDIA-Nemotron-Nano-9B-v2) on multi-step tool calling with **GRPO (Group Relative Policy Optimization)** against the **Workplace Assistant** environment — a simulated office (calendar, email, analytics, project management, CRM) that issues realistic enterprise tasks and verifies the model's tool calls against final database state.

The full series runs in about 3-5 hours and produces a checkpoint with measurably higher tool-calling accuracy on the [Berkeley Function Calling Leaderboard (BFCL) v3](https://gorilla.cs.berkeley.edu/leaderboard.html). You will validate the stack on a single 8×H100/A100 node first (3 GRPO steps, about 45 minutes), then submit a multi-node Slurm batch job for the production run. W&B `train:reward_mean` should trend up; `val:accuracy` should rise steadily.

For the fast path, jump to [Setup](/latest/train-models/nemo-rl-grpo/setup).

---

## Prerequisites

- **Hardware**: one or more nodes with 8× NVIDIA GPUs (80 GB each, H100 or A100); 64 GB+ RAM per node. Single-node validation needs 1 node; production training is best on 8+ nodes.
- **Storage**: 100 GB+ free on a shared filesystem.
- **Software**: Linux, Python 3.12+, Git, and Slurm for the multi-node run.
- **Familiarity**: Python and LLM fine-tuning. Deep RLVR/GRPO knowledge is not required.

NeMo Gym itself does not require GPUs — GPUs are only used by NeMo RL during training.

Optional accounts:

- **Weights & Biases (W&B)** for experiment tracking ([sign up](https://wandb.ai/signup), [get API key](https://wandb.ai/authorize)). Training proceeds without it.
- **HuggingFace** for model downloads ([create token](https://huggingface.co/settings/tokens)). Recommended to avoid rate limits.
